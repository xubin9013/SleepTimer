# SleepTimer 安装程序 — 构建说明

依据 `D:\ScreenOff\260714` 下的《样式方案》《交互逻辑》《要求》及 UI 图标生成的 Windows 定时熄屏客户端安装包。

## 技术栈
- **Tauri 2.x**：Rust 后端 + TypeScript/Vite 前端（复用系统 WebView2，不打包 Chromium）。
- 选择原因：需满足「安装包 ≤ 5MB」+「CSS 设计令牌 1:1 还原」+「原生能力（熄屏/自启/托盘/单实例/日志）」。Electron 自带内核 >80MB，直接出局。

## 交付物
| 文件 | 说明 | 体积 |
|------|------|------|
| `SleepTimer-Setup.exe` | NSIS 安装包，默认安装到 `D:\Program Files\SleepTimer` | **1.75 MB** ✅ < 5MB |
| `src-tauri/target/release/sleeptimer.exe`（安装后为 `SleepTimer.exe`） | 主程序（前端已嵌入，图标内嵌） | 3.63 MB |

## 安装后目录结构
```
D:\Program Files\SleepTimer\
├── SleepTimer.exe        # 主程序（图标已内嵌 exe 资源，安装目录无独立 ico 文件）
├── Uninstall.exe         # 卸载程序（卸载时删除整个安装目录）
├── config\               # 配置文件（app.json）
└── log\                  # 熄屏日志(off.log) + 调试日志(debug.log)，每 5MB 自动滚动分割
```
> 安装完成页含「立即运行 SleepTimer」复选框（默认勾选）。
> 安装首屏（目录选择之前）为「安装选项」页：可勾选「创建桌面快捷方式」「随系统启动（开机自动运行）」，两项均默认勾选。
> 若以标准用户身份运行导致 `Program Files` 不可写，配置/日志会自动回退到 `%LOCALAPPDATA%\SleepTimer`。

## 已实现功能（对照规范）
- **外壳**：自定义标题栏（图标/程序名/版本 `V1.0.260714`/当前执行方案 Pill/主题切换/一键熄屏/最小化/关闭）、可折叠侧边栏（方案管理/设置/熄屏日志）。
- **主题**：明/暗一键切换，CSS 变量设计令牌 1:1 还原，持久化。
- **方案管理**：新建（空/重名校验，错误以红色文字内联显示在输入框下方，名称限 10 字符）、**双击方案标签/名称弹出重命名弹窗**（样式同新建，限 10 字符）、时间点的添加/修改（三列滚轮选择器）/删除、删除方案三场景确认（普通/在循环/当前执行）。「添加时间」按钮与方案名称同行右侧。**点击方案标签仅切换查看的方案（本地 viewPlan），不同步顶部"当前执行"pill 与设置页的"执行方案"选择。**
- **设置**（各模块标题已取消）：「执行方案」行（胶囊芯片选择当前执行方案，适应长度显示完整名称）+ 分隔线 +「启用循环」行（子标题"多方案按序列循环执行"）；循环配置（天/月粒度、间隔、开始日期/月份选择器、循环顺序增删、时间轴预览）；随系统启动（注册表）、关闭最小化到托盘、熄屏并锁定、熄屏前倒计时（秒数 5~300，无上下箭头）；重置按钮置于右侧（与上方开关同款样式）。已移除「配置文件路径」设置项。
- **熄屏日志**：表格/分页（10/20/50/全部）/刷新动画/清空（二次确认）/空状态。
- **熄屏与倒计时**：`SendMessage(WM_SYSCOMMAND, SC_MONITORPOWER, 2)` 熄屏（+可选锁屏）；右下角深色固定倒计时通知，进度条同步，Esc/点击可取消；定时器每秒检查当前方案时间点触发。
- **原生能力**：系统托盘（显示/退出菜单）、单实例互斥（重复启动聚焦原窗口）、注册表自启动、debug 全程记录（启动→各操作→熄屏）。

## 已知简化（如需要可继续完善）
- 循环方案顺序用「添加/移除」按钮调整，未实现拖拽排序（规范要求 dnd-kit 类交互）。
- 时间滚轮为 scroll-snap 吸附式，未做无限循环滚动。
- 循环执行简化为「监听当前方案的时间点」；严格的按开始日期+间隔跨方案轮转计算未接入定时器。

## 如何重新构建
```bash
cd SleepTimer
npm install
# ⚠️ 必须用 cargo tauri build（不要用 cargo build --release），否则前端不会正确嵌入 exe
cargo tauri build --no-bundle   # 自动执行 npm run build + Rust 编译 + 前端嵌入
# 用 makensis 编译 NSIS 安装包（默认装到 D:\Program Files\SleepTimer）
"C:\Users\Administrator\AppData\Local\tauri\NSIS\makensis.exe" /V1 installer.nsi
```
> **重要**：`cargo tauri build` 会走完整 Tauri 管线（含 `beforeBuildCommand: npm run build` + 前端资源嵌入）。直接用 `cargo build --release` 会导致前端未嵌入，运行时出现 `ERR_CONNECTION_REFUSED`。
